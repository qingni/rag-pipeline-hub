## 1. 前端组件

### 1.1 组件结构

```
HybridSearch.vue (混合检索主组件)
├── SearchInput              # 查询输入
├── SearchResults            # 搜索结果（含 RRF/Reranker 分数）
└── SearchModeIndicator      # 检索模式指示器（hybrid/dense_only）

VectorSearch.vue (向量搜索组件)
├── SearchInput              # 查询输入
└── SearchResults            # 搜索结果
```

### 1.2 HybridSearch.vue

混合检索交互组件，提供：
- 查询文本输入
- 检索模式展示（hybrid / dense_only）
- RRF 分数和 Reranker 分数的结果展示
- Reranker 可用性状态指示

```vue
<template>
  <div class="hybrid-search">
    <div class="search-header">
      <h3>混合检索</h3>
      <SearchModeIndicator :mode="searchMode" :reranker-available="rerankerAvailable" />
    </div>
    
    <!-- 查询输入 -->
    <div class="search-input">
      <t-textarea
        v-model="queryText"
        placeholder="输入查询文本..."
        :maxlength="2000"
        :autosize="{ minRows: 2, maxRows: 6 }"
      />
      <div class="search-options">
        <t-select v-model="selectedCollection" placeholder="选择 Collection">
          <t-option v-for="col in collections" :key="col.name" :value="col.name" :label="col.name" />
        </t-select>
        <t-input-number v-model="topK" :min="1" :max="100" label="Top-K" />
        <t-switch v-model="enableReranker" label="Reranker 精排" />
      </div>
      <t-button theme="primary" :loading="isSearching" @click="handleSearch">
        搜索
      </t-button>
    </div>
    
    <!-- 搜索结果 -->
    <div v-if="searchResults.length" class="search-results">
      <div class="result-meta">
        <t-tag theme="primary">{{ searchMode }}</t-tag>
        <span>耗时: {{ queryTimeMs }}ms</span>
        <span>候选集: {{ totalCandidates }}</span>
      </div>
      <div v-for="(result, index) in searchResults" :key="index" class="result-card">
        <div class="result-scores">
          <span>RRF: {{ result.rrf_score?.toFixed(4) }}</span>
          <span v-if="result.reranker_score">Reranker: {{ result.reranker_score?.toFixed(4) }}</span>
          <span>最终: {{ result.final_score?.toFixed(4) }}</span>
        </div>
        <div class="result-text">{{ result.text }}</div>
        <div class="result-meta">
          <t-tag size="small">{{ result.doc_id }}</t-tag>
          <t-tag size="small" variant="outline">chunk #{{ result.chunk_index }}</t-tag>
        </div>
      </div>
    </div>
  </div>
</template>

<script setup>
import { ref, computed, onMounted } from 'vue'
import { useVectorIndexStore } from '@/stores/vectorIndexStore'

const store = useVectorIndexStore()

const queryText = ref('')
const selectedCollection = ref('')
const topK = ref(5)
const enableReranker = ref(true)

const isSearching = computed(() => store.isSearching)
const searchResults = computed(() => store.searchResults)
const searchMode = computed(() => store.searchMode)
const queryTimeMs = computed(() => store.queryTimeMs)
const totalCandidates = computed(() => store.totalCandidates)
const rerankerAvailable = computed(() => store.rerankerAvailable)
const collections = computed(() => store.collections)

async function handleSearch() {
  await store.hybridSearch({
    collection_name: selectedCollection.value,
    query_text: queryText.value,
    top_k: topK.value,
    enable_reranker: enableReranker.value
  })
}

onMounted(async () => {
  await store.checkRerankerHealth()
})
</script>
```

### 1.3 组件职责分配

| 组件 | 职责 | 状态来源 |
|------|------|----------|
| HybridSearch.vue | 混合检索交互、结果展示 | vectorIndexStore |
| VectorSearch.vue | 向量搜索交互 | vectorIndexStore |
| SearchInput | 查询文本输入 | props + emit |
| SearchResults | 搜索结果列表展示 | props |

## 2. 数据模型

### 2.1 存储概述

检索查询模块涉及的持久化数据：

| 存储层 | 用途 | 数据内容 |
|--------|------|----------|
| 关系数据库 | 查询历史 | query_history 表 |
| 关系数据库 | 统计更新 | index_statistics 表（搜索次数、耗时） |
| 文件系统 | 检索结果 | hybrid_search/ 目录下的 JSON 文件 |

### 2.2 QueryHistory 模型

```python
class QueryHistory(Base):
    """查询历史模型"""
    __tablename__ = "query_history"
    
    id = Column(String(36), primary_key=True)
    index_id = Column(String(36), ForeignKey("vector_indexes.id"), nullable=False)
    
    # 查询信息
    query_text = Column(String(2000), nullable=True, comment="查询文本")
    query_vector_hash = Column(String(64), nullable=True, comment="查询向量哈希")
    top_k = Column(Integer, nullable=False, comment="请求数量")
    threshold = Column(Float, nullable=True, comment="相似度阈值")
    
    # 结果信息
    result_count = Column(Integer, default=0, comment="返回结果数")
    search_time_ms = Column(Float, nullable=False, comment="搜索耗时(ms)")
    search_mode = Column(String(20), nullable=True, comment="检索模式(hybrid/dense_only)")
    
    # 时间戳
    created_at = Column(DateTime, default=datetime.utcnow, comment="创建时间")
    
    # 关系
    vector_index = relationship("VectorIndex", back_populates="query_history")
```

### 2.4 数据库表 DDL

```sql
-- 查询历史表
CREATE TABLE IF NOT EXISTS query_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    index_id INTEGER NOT NULL,
    query_text VARCHAR(2000),
    query_vector TEXT,
    top_k INTEGER NOT NULL,
    filters TEXT,
    results_count INTEGER DEFAULT 0,
    search_time_ms FLOAT NOT NULL,
    search_mode VARCHAR(20),
    response_time_ms FLOAT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (index_id) REFERENCES vector_indexes(id)
);

-- 查询历史索引
CREATE INDEX idx_query_history_index_id ON query_history(index_id);
CREATE INDEX idx_query_history_created_at ON query_history(created_at DESC);
```

## 3. 数据流

### 3.1 混合检索数据流

```
┌───────────────────┐
│  HybridSearch.vue │
│  (用户输入查询)     │
└────────┬──────────┘
         │ handleSearch()
         ↓
┌───────────────────┐
│ vectorIndexStore  │
│ hybridSearch()    │
└────────┬──────────┘
         │ POST /hybrid-search
         ↓
┌───────────────────┐
│   API Layer       │
│ vector_index.py   │
└────────┬──────────┘
         │
         ↓
┌───────────────────┐
│  Service Layer    │
│ hybrid_search()   │
├───────────────────┤
│ 1. BM25 encode   │ → bm25_service.encode_query()
│ 2. 双路召回+RRF   │ → milvus_provider.hybrid_search()
│ 3. Reranker 精排  │ → reranker_service.rerank()
│ 4. 记录查询历史   │ → query_history INSERT
└────────┬──────────┘
         │
         ↓
┌───────────────────┐
│  返回结果到前端    │
│  更新 store 状态   │
└───────────────────┘
```
