### 1.2 Search.vue

当前搜索页的真实入口是 `frontend/src/views/Search.vue`，它通过 `useSearchStore()` 管理状态，并组合 `SearchInput`、`SearchConfig`、`SearchResults` 与 `SearchHistory`。

页面当前提供：
- 查询文本输入与清空
- Collection / Index 选择
- Top-K、阈值、度量类型、Reranker 参数配置
- 检索模式、Reranker 可用性、阶段耗时展示
- 搜索历史查看、复用、删除与清空

```vue
<script setup>
import { ref, onMounted } from 'vue'
import SearchInput from '../components/Search/SearchInput.vue'
import SearchConfig from '../components/Search/SearchConfig.vue'
import SearchResults from '../components/Search/SearchResults.vue'
import SearchHistory from '../components/Search/SearchHistory.vue'
import { useSearchStore } from '../stores/searchStore'

const searchStore = useSearchStore()
const activeTab = ref('results')

onMounted(async () => {
  await Promise.all([
    searchStore.loadAvailableIndexes(),
    searchStore.loadAvailableCollections(),
    searchStore.loadRerankerHealth(),
    searchStore.loadHistory()
  ])
})
</script>
```

> 说明：检索服务内部支持 Query Enhancement，但当前搜索页前端并未直接展示 `query_enhancement` 详情，也未在 `searchStore` 中维护这块独立状态。

### 1.3 组件职责分配

| 组件 | 职责 | 状态来源 |
|------|------|----------|
| Search.vue | 页面布局、面板协调、Tab 切换 | searchStore |
| SearchConfig.vue | 搜索参数配置 | props + emit / searchStore |
| SearchInput | 查询文本输入 | props + emit |
| SearchResults | 搜索结果列表展示 | props |
| SearchHistory | 搜索历史展示与操作 | props + emit |
| SearchModeIndicator | 检索模式和 Reranker 状态指示 | props |

### 1.4 文本生成页对检索模块的复用

除独立的 `Search.vue` 搜索页面外，文本生成页也会直接复用搜索 API，但消费方式更轻量：

- 使用 `getAvailableCollections()` 获取可选知识库列表
- 使用 `executeHybridSearch()` 获取生成前上下文
- 不直接展示 RRF / Reranker 调试信息，而是将结果映射为生成上下文
- 将 `source_document`、`similarity_score` 等字段进一步映射为引用来源列表

典型映射逻辑如下：

```javascript
const contextItems = response.data.results.map((result, index) => ({
  content: result.text_content || result.content || result.text || '',
  source_file: result.source_document || result.document_name || result.metadata?.source || `来源 ${index + 1}`,
  similarity: result.similarity_score || result.score || result.similarity || 0,
  chunk_id: result.chunk_id || null,
  metadata: result.metadata
}))
```

这意味着检索模块既服务“搜索页”，也服务“生成页的自动召回链路”。

### 2.2 Pydantic Schema 模型

#### 2.2.1 请求模型

**SearchRequest**

```python
class SearchRequest(BaseModel):
    query_text: str          # 查询文本（1-2000字符）
    index_ids: List[str]     # 目标索引ID列表（可选）
    top_k: int = 10          # 返回结果数量（1-100）
    threshold: float = 0.5   # 相似度阈值（0-1）
    metric_type: str = "cosine"  # 相似度计算方法
```

**HybridSearchRequest**

```python
class HybridSearchRequest(BaseModel):
    query_text: str                    # 查询文本（1-2000字符）
    collection_ids: List[str] = None   # 目标 Collection ID 列表（最多5个）
    top_k: int = 10                    # 最多返回结果数量（1-100）
    threshold: float = 0.5             # 相似度阈值（0-1）
    metric_type: str = "cosine"        # 相似度计算方法
    search_mode: str = "auto"          # 检索模式（auto/hybrid/dense_only）
    reranker_top_n: int = 20           # Reranker 候选集大小（10-100）
    reranker_top_k: int = None         # Reranker 最大返回数（默认使用 top_k）
    enable_query_enhancement: bool = True  # 🆕 是否启用查询增强
    page: int = 1                      # 🆕 分页页码
    page_size: int = 10                # 🆕 每页数量（1-100）
```

## 3. 数据流

### 3.1 混合检索数据流

```
┌───────────────────┐
│    Search.vue     │
│  (用户输入查询)     │
└────────┬──────────┘
         │ handleSearch()
         ↓
┌───────────────────┐
│   searchStore     │
│    search()       │
└────────┬──────────┘
         │ POST /api/v1/search/hybrid
         ↓
┌───────────────────┐
│   API Layer       │
│ search.py         │
└────────┬──────────┘
         │
         ↓
┌───────────────────────────────────────────────────┐
│  Service Layer - search_service.hybrid_search()   │
├───────────────────────────────────────────────────┤
│ 0. 查询增强  🆕  │ → query_enhancement_service    │
│    (Query Rewrite │   .enhance_query()             │
│     + Multi-query)│   → rewritten_query            │
│                   │   + sub_queries                │
│ 1. Embedding      │ → embedding_service            │
│    (支持跨模型     │   ._embed_query()              │
│     Multi-query   │   → 各变体并行嵌入             │
│     并行嵌入)      │                                │
│ 2. BM25 encode    │ → bm25_service.encode_query()  │
│ 3. 双路召回+RRF   │ → milvus_provider              │
│    (+Multi-query  │   .hybrid_search()             │
│     并行检索合并)  │                                │
│ 4. 三层防御  🆕   │                                │
│    ├ 候选配额控制  │                                │
│    ├ 近重复去重    │                                │
│    ├ Reranker精排  │ → reranker_service.rerank()    │
│    ├ 动态阈值过滤  │                                │
│    └ 多样性控制    │                                │
│ 5. 记录查询历史   │ → search_history INSERT         │
│ 6. 附加增强信息   │ → query_enhancement info       │
└────────┬──────────────────────────────────────────┘
         │
         ↓
┌───────────────────┐
│  返回结果到前端    │
│  更新 store 状态   │
│  - results         │
│  - searchTiming    │
│  - totalCount      │
│  - searchMode      │
└───────────────────┘
```

```
└────────────────────────────┘
```

### 3.3 文本生成页的数据流

```text
Generation.vue
  -> generationStore.loadAvailableCollections()
     -> searchApi.getAvailableCollections()

Generation.vue
  -> generationStore.generateStream(question)
     -> generationStore.retrieveContext(question)
        -> searchApi.executeHybridSearch({
             query_text,
             collection_ids,
             top_k,
             threshold: 0.3
           })
        -> 映射为 context / sources
     -> generationApi.generateTextStream({
          question,
          model,
          max_tokens,
          context
        })
```

在这个链路里，检索模块返回的是“原始检索结果”，生成模块只抽取其中与回答生成直接相关的字段。
