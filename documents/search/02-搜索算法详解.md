## 1. 语义搜索原理

### 1.1 什么是语义搜索

语义搜索是一种基于文本语义理解的搜索方式，与传统的关键词匹配不同，它能够理解查询的含义，找到语义相似的内容，即使它们不包含相同的关键词。

### 1.2 工作流程

```
用户查询文本
     │
     ▼
┌─────────────────┐
│  文本向量化      │  ← Embedding 模型
│  (Query → Vector)│
└─────────────────┘
     │
     ▼
┌─────────────────┐
│  向量相似度搜索  │  ← 向量数据库
│  (ANN Search)   │
└─────────────────┘
     │
     ▼
┌─────────────────┐
│  结果排序过滤    │  ← 阈值、Top-K
└─────────────────┘
     │
     ▼
返回相似文档
```

### 1.3 核心概念

| 概念 | 说明 |
|------|------|
| **Embedding** | 将文本转换为高维向量的过程 |
| **向量空间** | 所有向量所在的高维空间 |
| **相似度** | 衡量两个向量之间相似程度的指标 |
| **ANN** | 近似最近邻搜索 (Approximate Nearest Neighbor) |

### 1.4 语义搜索 vs 关键词搜索

| 特性 | 语义搜索 | 关键词搜索 |
|------|---------|-----------|
| 匹配方式 | 语义相似 | 词汇匹配 |
| 同义词处理 | 自动理解 | 需要配置 |
| 上下文理解 | 支持 | 不支持 |
| 计算复杂度 | 较高 | 较低 |
| 适用场景 | 自然语言查询 | 精确关键词查询 |

## 2. 相似度计算方法

### 2.1 余弦相似度 (Cosine Similarity)

**原理**: 计算两个向量夹角的余弦值，衡量方向的相似性。

**公式**:
$$\cos(\theta) = \frac{A \cdot B}{||A|| \times ||B||} = \frac{\sum_{i=1}^{n}A_i B_i}{\sqrt{\sum_{i=1}^{n}A_i^2} \times \sqrt{\sum_{i=1}^{n}B_i^2}}$$

**特点**:
- 值域: [-1, 1]
- 1 表示完全相同方向
- 0 表示正交（无关）
- -1 表示完全相反方向

**代码实现**:
```python
import numpy as np

def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:
    """计算余弦相似度"""
    dot_product = np.dot(a, b)
    norm_a = np.linalg.norm(a)
    norm_b = np.linalg.norm(b)
    return dot_product / (norm_a * norm_b)
```

**适用场景**:
- ✅ 文本语义匹配
- ✅ 文档相似度计算
- ✅ 推荐系统

### 2.2 欧氏距离 (Euclidean Distance / L2)

**原理**: 计算两个向量在欧几里得空间中的直线距离。

**公式**:
$$d(A, B) = \sqrt{\sum_{i=1}^{n}(A_i - B_i)^2}$$

**特点**:
- 值域: [0, ∞)
- 0 表示完全相同
- 值越大表示越不相似

**代码实现**:
```python
def euclidean_distance(a: np.ndarray, b: np.ndarray) -> float:
    """计算欧氏距离"""
    return np.linalg.norm(a - b)

def euclidean_similarity(a: np.ndarray, b: np.ndarray) -> float:
    """将欧氏距离转换为相似度 (0-1)"""
    distance = euclidean_distance(a, b)
    return 1 / (1 + distance)
```

**适用场景**:
- ✅ 图像特征匹配
- ✅ 聚类分析
- ✅ 异常检测

### 2.3 点积 (Inner Product / Dot Product)

**原理**: 计算两个向量的点积，同时考虑方向和大小。

**公式**:
$$A \cdot B = \sum_{i=1}^{n}A_i \times B_i$$

**特点**:
- 值域: (-∞, +∞)
- 正值表示方向相近
- 负值表示方向相反
- 受向量长度影响

**代码实现**:
```python
def dot_product(a: np.ndarray, b: np.ndarray) -> float:
    """计算点积"""
    return np.dot(a, b)
```

**适用场景**:
- ✅ 归一化向量的相似度计算
- ✅ 注意力机制
- ✅ 向量已预归一化的场景

### 2.4 相似度方法对比

| 方法 | 值域 | 是否需要归一化 | 计算复杂度 | 推荐场景 |
|------|------|--------------|-----------|----------|
| 余弦相似度 | [-1, 1] | 否 | O(n) | 文本语义 |
| 欧氏距离 | [0, ∞) | 建议 | O(n) | 图像特征 |
| 点积 | (-∞, +∞) | 是 | O(n) | 归一化向量 |

### 2.5 分数转换

由于不同度量类型的原始分数值域不同，需要统一转换到 [0, 1] 范围：

```python
def normalize_score(score: float, metric_type: str) -> float:
    """将原始分数转换为 0-1 范围的相似度"""
    if metric_type == "cosine" or metric_type == "dot_product":
        # 内积结果，假设向量已归一化，值域 [-1, 1]
        return (score + 1) / 2
    elif metric_type == "euclidean":
        # 欧氏距离，值域 [0, ∞)
        return 1 / (1 + score)
    else:
        return score
```

## 3. 阈值过滤策略

### 3.1 阈值的作用

阈值用于过滤掉相似度较低的结果，确保返回的结果具有足够的相关性。

### 3.2 阈值设置建议

| 阈值范围 | 含义 | 适用场景 |
|---------|------|----------|
| 0.9 - 1.0 | 高度相似 | 精确匹配、去重 |
| 0.7 - 0.9 | 较为相似 | 语义搜索、问答 |
| 0.5 - 0.7 | 中等相似 | 推荐、探索 |
| 0.3 - 0.5 | 略有相似 | 宽泛搜索 |
| 0.0 - 0.3 | 几乎不相似 | 一般不使用 |

### 3.3 动态阈值策略

```python
def get_dynamic_threshold(query_length: int, result_count: int) -> float:
    """根据查询长度和结果数量动态调整阈值"""
    base_threshold = 0.5
    
    # 短查询降低阈值（更宽泛）
    if query_length < 10:
        base_threshold -= 0.1
    elif query_length > 100:
        base_threshold += 0.1
    
    # 结果太少时降低阈值
    if result_count < 3:
        base_threshold -= 0.1
    
    return max(0.3, min(0.9, base_threshold))
```

### 3.4 阈值过滤实现

```python
def filter_by_threshold(
    results: List[SearchResult], 
    threshold: float
) -> List[SearchResult]:
    """根据阈值过滤搜索结果"""
    return [r for r in results if r.score >= threshold]
```

## 4. 多索引搜索与结果合并

### 4.1 多索引搜索场景

- 搜索多个文档的索引
- 搜索不同时间段的索引
- 搜索不同类型的内容

### 4.2 搜索流程

```
用户查询
    │
    ▼
┌─────────────────────────────────────┐
│           并行搜索多个索引            │
├─────────┬─────────┬─────────────────┤
│ 索引 A  │ 索引 B  │     索引 C      │
│ Top-K   │ Top-K   │     Top-K       │
└────┬────┴────┬────┴────────┬────────┘
     │         │             │
     └─────────┼─────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│           结果合并与排序              │
│  1. 收集所有结果                      │
│  2. 按相似度排序                      │
│  3. 去重（可选）                      │
│  4. 截取最终 Top-K                   │
└─────────────────────────────────────┘
               │
               ▼
         返回最终结果
```

### 4.3 结果合并策略

#### 策略1: 简单合并排序

```python
def merge_results_simple(
    results_list: List[List[SearchResult]],
    top_k: int
) -> List[SearchResult]:
    """简单合并：收集所有结果，按分数排序"""
    all_results = []
    for results in results_list:
        all_results.extend(results)
    
    # 按分数降序排序
    all_results.sort(key=lambda x: x.score, reverse=True)
    
    return all_results[:top_k]
```

#### 策略2: 加权合并

```python
def merge_results_weighted(
    results_list: List[List[SearchResult]],
    weights: List[float],
    top_k: int
) -> List[SearchResult]:
    """加权合并：不同索引的结果有不同权重"""
    all_results = []
    
    for results, weight in zip(results_list, weights):
        for result in results:
            result.score *= weight
        all_results.extend(results)
    
    all_results.sort(key=lambda x: x.score, reverse=True)
    return all_results[:top_k]
```

#### 策略3: 去重合并

```python
def merge_results_dedupe(
    results_list: List[List[SearchResult]],
    top_k: int,
    similarity_threshold: float = 0.95
) -> List[SearchResult]:
    """去重合并：移除重复或高度相似的结果"""
    all_results = []
    for results in results_list:
        all_results.extend(results)
    
    all_results.sort(key=lambda x: x.score, reverse=True)
    
    # 去重
    unique_results = []
    for result in all_results:
        is_duplicate = False
        for unique in unique_results:
            if text_similarity(result.text, unique.text) > similarity_threshold:
                is_duplicate = True
                break
        if not is_duplicate:
            unique_results.append(result)
    
    return unique_results[:top_k]
```

### 4.4 分数归一化

不同索引可能使用不同的度量类型，需要归一化分数：

```python
def normalize_scores_across_indexes(
    results_list: List[List[SearchResult]],
    metric_types: List[str]
) -> List[List[SearchResult]]:
    """跨索引归一化分数"""
    normalized_list = []
    
    for results, metric_type in zip(results_list, metric_types):
        normalized = []
        for result in results:
            result.score = normalize_score(result.score, metric_type)
            normalized.append(result)
        normalized_list.append(normalized)
    
    return normalized_list
```

### 5.2 结果优化

#### 重排序 (Re-ranking)

```python
def rerank_results(
    query: str,
    results: List[SearchResult],
    reranker_model
) -> List[SearchResult]:
    """使用重排序模型优化结果排序"""
    pairs = [(query, r.text) for r in results]
    scores = reranker_model.predict(pairs)
    
    for result, score in zip(results, scores):
        result.rerank_score = score
    
    results.sort(key=lambda x: x.rerank_score, reverse=True)
    return results
```

#### 多样性优化

```python
def diversify_results(
    results: List[SearchResult],
    top_k: int,
    diversity_factor: float = 0.3
) -> List[SearchResult]:
    """增加结果多样性，避免过于相似的结果"""
    diverse_results = [results[0]]
    
    for result in results[1:]:
        if len(diverse_results) >= top_k:
            break
        
        # 计算与已选结果的最大相似度
        max_sim = max(
            text_similarity(result.text, r.text) 
            for r in diverse_results
        )
        
        # 如果相似度低于阈值，加入结果
        if max_sim < (1 - diversity_factor):
            diverse_results.append(result)
    
    return diverse_results
```

### 5.3 性能优化

#### 缓存策略

```python
from functools import lru_cache

@lru_cache(maxsize=1000)
def cached_embed_query(query: str, model_name: str) -> List[float]:
    """缓存查询向量化结果"""

class SearchCache:
    def __init__(self, max_size: int = 100):
        self.cache = {}
        self.max_size = max_size
    
    def get_cache_key(self, query: str, index_ids: List[str], top_k: int) -> str:
        return f"{hash(query)}_{','.join(sorted(index_ids))}_{top_k}"
    
    def get(self, key: str) -> Optional[List[SearchResult]]:
        return self.cache.get(key)
    
    def set(self, key: str, results: List[SearchResult]):
        if len(self.cache) >= self.max_size:
            # LRU 淘汰
            oldest_key = next(iter(self.cache))
            del self.cache[oldest_key]
        self.cache[key] = results
```

## 6. 最佳实践

### 6.1 查询建议

1. **查询长度**: 建议 10-100 字符，过短语义不明确，过长噪音多
2. **查询质量**: 使用完整的句子或问题，避免单个词
3. **查询预处理**: 去除特殊字符、规范化空格

### 6.2 参数调优

| 参数 | 推荐值 | 调优建议 |
|------|--------|----------|
| Top-K | 10-20 | 根据结果质量调整 |
| 阈值 | 0.5-0.7 | 根据召回率/精确率平衡 |
| 多样性因子 | 0.2-0.4 | 根据结果重复度调整 |

### 6.3 监控指标

```python
class SearchMetrics:
    """搜索质量监控"""
    
    def __init__(self):
        self.total_searches = 0
        self.avg_result_count = 0
        self.avg_top_score = 0
        self.avg_latency_ms = 0
    
    def record(self, results: List[SearchResult], latency_ms: float):
        self.total_searches += 1
        
        if results:
            self.avg_result_count = (
                self.avg_result_count * (self.total_searches - 1) + len(results)
            ) / self.total_searches
            
            self.avg_top_score = (
                self.avg_top_score * (self.total_searches - 1) + results[0].score
            ) / self.total_searches
        
        self.avg_latency_ms = (
            self.avg_latency_ms * (self.total_searches - 1) + latency_ms
        ) / self.total_searches
```

### 6.4 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 结果不相关 | 查询太短/模糊 | 扩展查询、使用更具体的描述 |
| 结果太少 | 阈值过高 | 降低阈值、增加 Top-K |
| 结果重复 | 未去重 | 启用去重策略 |
| 搜索慢 | 索引未优化 | 使用 HNSW 等快速索引 |
| 分数异常 | 度量类型不匹配 | 检查索引度量类型配置 |
