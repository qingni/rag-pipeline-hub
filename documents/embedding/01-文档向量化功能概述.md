## 1. 功能概述

### 1.1 核心功能
文档向量化功能是RAG (Retrieval-Augmented Generation) 系统的核心组件，负责将文档分块后的文本内容转换为高维向量表示，为后续的向量索引和语义检索提供基础数据。

### 1.2 主要特性
- ✅ **多模型支持**: 支持BGE-M3、Qwen3、混元、Jina等主流Embedding模型
- ✅ **批量处理**: 支持单文本和批量文本向量化
- ✅ **分块集成**: 直接从分块结果进行向量化
- ✅ **智能重试**: 指数退避重试机制，处理API限流和超时
- ✅ **部分成功**: 批量处理支持部分成功，失败项可重试
- ✅ **结果持久化**: 向量结果保存为JSON格式，支持后续索引
- ✅ **详细日志**: 完整的请求日志和性能监控

### 1.3 设计原则
1. **可扩展性**: 基于OpenAI兼容API，易于接入新模型
2. **容错性**: 多层级错误处理和重试机制
3. **可观测性**: 详细的日志记录和性能指标
4. **高性能**: 批量处理、异步执行、结果缓存

## 5. 工作流程

### 5.1 完整向量化流程

```
1. 用户选择文档分块结果
   └─> DocumentSelector.vue
       └─> 获取已完成的分块结果列表

2. 用户选择Embedding模型
   └─> ModelSelector.vue
       └─> 显示可用模型及其参数

3. 配置向量化参数
   └─> EmbeddingConfiguration.vue
       └─> 设置批量大小、重试次数等

4. 执行向量化
   └─> embedding_service.embed_chunking_result()
       ├─> 加载分块结果中的所有chunks
       ├─> 调用模型API进行向量化
       ├─> 处理失败项和重试
       └─> 返回向量化结果

5. 保存结果
   └─> 保存到数据库 (embedding_results)
   └─> 保存JSON文件 (results/embedding/)

6. 前端展示结果
   └─> EmbeddingResults.vue
       ├─> 显示成功/失败统计
       ├─> 显示向量维度信息
       └─> 提供下载功能
```

### 5.2 错误处理流程

```
API调用失败
  │
  ▼
检查错误类型
  │
  ├─ RateLimitError (限流)
  │  └─> 指数退避重试，最多3次
  │
  ├─ APITimeoutError (超时)
  │  └─> 指数退避重试，最多3次
  │
  ├─ NetworkError (网络)
  │  └─> 指数退避重试，最多3次
  │
  ├─ AuthenticationError (认证)
  │  └─> 直接失败，不重试
  │
  └─ InvalidTextError (文本无效)
     └─> 记录失败，继续处理其他文本
```

## 6. 关键特性

### 6.1 多模型支持
系统支持多种主流Embedding模型，通过OpenAI兼容API统一调用：

| 模型 | 维度 | 特点 |
|------|------|------|
| bge-m3 | 1024 | 多语言支持，性能优秀 |
| qwen3-embedding-8b | 4096 | 高维度，语义理解强 |
| hunyuan-embedding | 1024 | 腾讯混元，中文优化 |
| jina-embeddings-v4 | 2048 | 多语言，高性能 |

### 6.2 智能重试机制
- **指数退避**: 初始延迟1秒，最大延迟32秒
- **可重试错误**: 限流、超时、网络错误
- **不可重试错误**: 认证失败、文本无效

### 6.3 批量处理
- **最大批量**: 1000条文本/批次
- **部分成功**: 支持部分成功，失败项单独记录
- **失败重试**: 失败项可单独重试

### 6.4 结果持久化
- **数据库记录**: 向量化任务元数据
- **JSON文件**: 完整向量数据和元数据
