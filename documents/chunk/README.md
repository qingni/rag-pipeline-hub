## 📊 关键数据

### 项目规模
- **后端文件**: 14个核心文件
- **前端组件**: 7个主要组件
- **API端点**: 15+个接口
- **数据表**: 4个核心实体

### 功能指标
- **支持策略**: 4种分块策略
- **支持格式**: PDF, DOCX, TXT, MD
- **最大文档**: 50MB
- **版本管理**: 智能覆盖/多版本并存
- **性能目标**: 
  - 标准文档(10k字符) < 5秒
  - 大型文档(50k字符) < 30秒
  - API响应 < 1秒

### 技术栈
- **后端**: Python 3.11+, FastAPI 0.104, SQLAlchemy 2.0
- **前端**: Vue 3.3, Pinia 2.1, TDesign 1.13
- **数据库**: SQLite/PostgreSQL
- **存储**: 数据库 + JSON文件

## 🚀 开始使用

### 开发者快速入门

1. **克隆代码**
   ```bash
   git clone <repository-url>
   cd rag-framework-spec
   git checkout 002-doc-chunking
   ```

2. **阅读文档**
   - 先阅读[功能概述](./01-文档分块功能概述.md)
   - 根据角色选择对应文档

3. **启动项目**
   ```bash
   # 后端
   cd backend
   pip install -r requirements.txt
   python -m src.main

   # 前端
   cd frontend
   npm install
   npm run dev
   ```

4. **访问应用**
   - 前端: http://localhost:5173/chunking
   - 后端: http://localhost:8000/docs
