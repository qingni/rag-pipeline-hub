## 📊 关键数据

### 项目规模
- **后端文件**: 20+个核心文件
- **前端组件**: 15+个主要组件（含可视化）
- **API端点**: 20+个接口
- **数据表**: 5个核心实体

### 功能指标
- **支持策略**: 6种分块策略（character, paragraph, heading, semantic, parent_child, hybrid）
- **支持格式**: PDF, DOCX, TXT, MD, XLSX, PPTX, HTML, JSON等
- **内容类型**: 文本、表格、图片、代码
- **最大文档**: 50MB
- **版本管理**: 智能覆盖/多版本并存
- **语义分块**: 支持Embedding增强（bge-m3, qwen3-embedding-8b）
- **性能目标**: 
  - 标准文档(10k字符) < 5秒
  - 大型文档(50k字符) < 30秒
  - API响应 < 1秒

### 技术栈
- **后端**: Python 3.11+, FastAPI 0.104, SQLAlchemy 2.0
- **前端**: Vue 3.3, Pinia 2.1, TDesign 1.13
- **数据库**: SQLite/PostgreSQL
- **存储**: 数据库 + JSON文件

## 🚀 开始使用

### 开发者快速入门

1. **克隆代码**
   ```bash
   git clone <repository-url>
   cd rag-pipeline-hub
   git checkout 002-doc-chunking
   ```

2. **阅读文档**
   - 先阅读[功能概述](./01-文档分块功能概述.md)
   - 根据角色选择对应文档

3. **启动项目**
   ```bash
   # 后端
   cd backend
   pip install -r requirements.txt
   python -m src.main

   # 前端
   cd frontend
   npm install
   npm run dev
   ```

4. **访问应用**
   - 前端: http://localhost:5173/chunking
   - 后端: http://localhost:8000/docs
