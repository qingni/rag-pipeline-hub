## 2. 核心组件详解

### 2.1 DocumentSelector.vue (文档选择器)

**功能**:
1. 加载已解析的文档列表
2. 支持搜索和格式过滤
3. 分页显示
4. 选择文档后自动加载最近分块结果

**关键特性**:
- 🔍 搜索防抖: 300ms
- 📄 自动选择: 首次加载自动选择第一个文档
- 🔄 智能加载: 选择文档后自动加载最近分块结果

**数据流**:
```
用户操作 → 触发loadDocuments() 
         → chunkingStore.loadParsedDocuments()
         → API: GET /chunking/documents/parsed
         → 更新parsedDocuments列表
         → 自动选择第一个文档(如果有)
         → 触发handleSelect()
         → chunkingStore.selectDocument()
         → 自动加载该文档最近分块结果
```

**主要方法**:
- `loadDocuments()`: 加载文档列表
- `handleSelect(docId)`: 选择文档
- `handleFilterChange()`: 过滤条件变化(防抖)
- `handlePageChange()`: 翻页

### 2.2 StrategySelector.vue (策略选择器)

**功能**:
1. 显示可用的6种分块策略
2. 支持策略描述和参数预览
3. 标记需要文档结构的策略
4. 集成策略推荐功能

**支持的策略**:
| 策略ID | 名称 | 描述 |
|--------|------|------|
| character | 按字数分块 | 按固定字符数切分文本 |
| paragraph | 按段落分块 | 按段落边界切分，合并小段落 |
| heading | 按标题分块 | 按Markdown标题层级切分 |
| semantic | 按语义分块 | 基于语义相似度智能切分，支持Embedding |
| parent_child | 父子分块 | 两层结构，父块提供上下文，子块用于检索 |
| hybrid | 混合分块 | 针对不同内容类型应用最合适的策略 |

**数据流**:
```
组件挂载 → loadStrategies()
        → chunkingStore.loadStrategies()
        → API: GET /chunking/strategies
        → 更新strategies列表

用户选择 → handleSelect(strategyType)
        → 查找strategy对象
        → chunkingStore.selectStrategy(strategy)
        → 触发参数初始化
```

**策略展示格式**:
```vue
<t-radio value="parent_child">
  <div class="strategy-item">
    <div class="strategy-header">
      <span>父子分块</span>
      <t-tag theme="success">推荐</t-tag>
    </div>
    <div class="strategy-desc">
      两层结构：父块提供上下文，子块用于检索
    </div>
  </div>
</t-radio>
```

**块数预估算法**:
```javascript
function estimateChunkCount() {
  const doc = chunkingStore.selectedDocument
  const estimatedTextLength = doc.size_bytes * 2  // 粗略估计
  
  if (strategyType === 'character') {
    const chunkSize = parameters.chunk_size || 500
    const overlap = parameters.overlap || 50
    const effectiveSize = chunkSize - overlap
    return Math.ceil(estimatedTextLength / effectiveSize)
  } else if (strategyType === 'paragraph') {
    const avgSize = (parameters.min_chunk_size + parameters.max_chunk_size) / 2
    return Math.ceil(estimatedTextLength / avgSize)
  } else if (strategyType === 'parent_child') {
    const childSize = parameters.child_chunk_size || 500
    return Math.ceil(estimatedTextLength / childSize)
  } else {
    // 对于heading、semantic、hybrid较难预估
    return Math.ceil(estimatedTextLength / 800)
  }
}
```

## 5. 工作流程

### 5.1 完整分块流程

```
1. 用户进入分块页面
   └─> DocumentChunk.vue mounted
       └─> 加载文档列表
       └─> 加载策略列表（6种策略）

2. 用户选择文档
   └─> handleSelect(docId)
       └─> chunkingStore.selectDocument(doc)
           ├─> 更新selectedDocument
           └─> loadLatestResultForDocument(doc.id)
               └─> API: GET /result/latest/{doc.id}
                   └─> 如果有结果，加载显示

3. 用户选择策略
   └─> handleSelect(strategyType)
       └─> chunkingStore.selectStrategy(strategy)
           ├─> 更新selectedStrategy
           ├─> 初始化strategyParameters
           └─> loadLatestResultForDocument(doc.id, strategy.type)

4. 用户配置参数
   └─> handleParamChange()
       └─> chunkingStore.updateParameters(params)
           └─> 更新strategyParameters

5. 用户点击开始分块
   └─> handleChunk()
       └─> chunkingStore.createChunkingTask()
           ├─> API: POST /chunk
           ├─> 创建任务
           ├─> 开始轮询
           └─> startTaskPolling()
               └─> 每2秒调用 updateTaskStatus()
                   └─> API: GET /task/{task_id}
                       └─> 如果完成，自动加载结果

6. 显示分块结果
   └─> loadChunkingResult(result_id) 或 loadParentChildResult(result_id)
       └─> API: GET /result/{result_id}
           ├─> 更新currentResult
           ├─> 更新chunks列表
           ├─> 更新parentChunks列表（父子分块）
           └─> 自动选择第一个chunk显示

7. 可视化展示
   └─> ChunkVisualizer 根据策略选择可视化组件
       ├─> parent_child → TreeVisualizer
       ├─> hybrid → HybridVisualizer
       └─> 其他 → LinearVisualizer
```

### 5.2 智能结果加载流程

```
选择文档
  │
  ▼
loadLatestResultForDocument(docId)
  │
  ├─ 查询: 文档的任意最近结果
  │  └─> 如果有: 加载显示
  │
  ▼
选择策略
  │
  ▼
loadLatestResultForDocument(docId, strategyType)
  │
  ├─ 查询: 文档 + 该策略的最近结果
  │  └─> 如果有: 加载显示
  │
  ▼
修改参数 (不自动加载，避免频繁请求)
  │
  ▼
点击"开始分块"创建新任务
```

### 5.3 版本管理流程

```
创建任务时
  │
  ▼
检查已有结果
  │
  ├─ 没有结果
  │  └─> 创建版本1
  │
  ├─ 有结果，参数完全相同
  │  └─> 直接返回已有结果
  │
  ├─ 有结果，overwrite_mode="auto"
  │  ├─ 参数微调(如overlap: 100→50)
  │  │  └─> 覆盖旧结果(is_active=false)
  │  │  └─> 创建新版本(version++)
  │  │
  │  └─ 参数大改(如chunk_size: 500→2000)
  │     └─> 保留旧结果
  │     └─> 创建新版本
  │
  ├─ overwrite_mode="always"
  │  └─> 总是覆盖
  │
  └─ overwrite_mode="never"
     └─> 总是新建版本
```

### 5.4 父子分块结果加载流程

```
API: GET /result/{result_id}
  │
  ▼
检测策略类型
  │
  ├─ strategy_type !== 'parent_child'
  │  └─> 返回标准结果（chunks数组）
  │
  └─> strategy_type === 'parent_child'
      │
      ▼
   加载父块和子块
      ├─> parent_chunks: 从ParentChunk表查询
      ├─> chunks: 从Chunk表查询（带parent_id）
      │
      ▼
   组装层级结构
      └─> 返回 {
            parent_chunks: [...],
            chunks: [...],
            statistics: {
              parent_count: 5,
              child_count: 20,
              avg_children_per_parent: 4.0
            }
          }
```

### 5.5 混合分块结果处理流程

```
API: GET /result/{result_id}
  │
  ▼
检测策略类型
  │
  └─> strategy_type === 'hybrid'
      │
      ▼
   加载所有类型块
      ├─> 从Chunk表查询（带chunk_type）
      │
      ▼
   按类型分组统计
      └─> chunk_type_distribution: {
            text: 25,
            table: 5,
            image: 3,
            code: 2
          }
      │
      ▼
   返回分块列表
      └─> chunks数组（每个带chunk_type字段）
```
