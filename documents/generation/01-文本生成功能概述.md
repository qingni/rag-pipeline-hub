## 1. 功能概述

### 1.1 核心功能
文本生成功能是RAG (Retrieval-Augmented Generation) 系统的最终输出环节，负责接收用户问题和检索到的上下文，调用大语言模型生成智能回答。它将搜索模块检索到的相关文档片段作为上下文，结合用户问题构建Prompt，通过LLM生成准确、有据可查的回答。

### 1.2 主要特性
- ✅ **RAG集成**: 自动从 Collection（逻辑知识库）执行混合检索，获取相关文档作为上下文
- ✅ **多模型支持**: 支持DeepSeek V3.2、DeepSeek V3.1等模型
- ✅ **流式输出**: 后端输出 SSE 格式数据，前端通过 `fetch + ReadableStream` 实时展示生成内容
- ✅ **参数配置**: 可配置温度、最大Token数等参数
- ✅ **引用标注**: 自动标注回答中的引用来源
- ✅ **上下文管理**: 支持多 Collection 检索、Top-K配置、Token预算截断
- ✅ **Markdown 渲染**: 结果支持标题、列表、代码块、表格等富文本展示，并进行安全清洗
- ✅ **历史记录**: 自动保存生成历史，支持查看、单条删除和全部清空
- ✅ **取消生成**: 支持中断正在进行的生成请求
- ✅ **失败重试**: 生成失败后可基于当前问题和配置一键重试

### 1.3 设计原则
1. **检索增强**: 基于检索结果生成，提高回答准确性
2. **可追溯**: 回答带有引用标注，便于验证
3. **用户友好**: 流式输出提供即时反馈
4. **灵活配置**: 支持多种模型和参数调整

## 5. 工作流程

### 5.1 RAG生成流程

```
1. 用户输入问题
   └─> GenerationInput.vue
       └─> 支持多行输入

2. 配置生成参数
   └─> GenerationConfig.vue
       ├─> 选择LLM模型
       ├─> 设置温度参数
       ├─> 设置最大Token
       ├─> 选择知识库 Collection（可多选）
       └─> 设置Top-K

3. 检索相关文档（自动）
   └─> generationStore.retrieveContext()
       ├─> 调用混合检索 API (`executeHybridSearch`)
       ├─> 基于 Collection 获取 Top-K 相关文档
       └─> 构建context数组

4. 执行生成
   └─> generation_service.generate_stream()
       ├─> 验证请求参数
       ├─> 按 token 预算截断上下文
       ├─> 构建Prompt（系统提示+上下文+问题）
       ├─> 调用LLM（流式）
       ├─> 逐块返回内容
       └─> 保存历史记录

5. 展示结果
   └─> GenerationResult.vue
       ├─> 流式显示生成内容
       ├─> Markdown渲染 + 安全清洗
       └─> 引用标注高亮 [1], [2]...

6. 展示引用来源
   └─> SourceReference.vue
       ├─> 显示Top-3引用
       ├─> 显示原文内容
       └─> 显示相似度分数
```

### 5.2 流式生成流程

```
前端发起 POST 请求（fetch）
  │
  ▼
┌─────────────────────────────────────┐
│   POST /api/v1/generation/stream    │
│  ├─ question: "用户问题"            │
│  ├─ model: "deepseek-v3.2"           │
│  ├─ temperature: 0.7               │
│  ├─ max_tokens: 4096               │
│  └─ context: [检索到的文档]         │
└─────────────────────────────────────┘
  │
  ▼
构建Prompt
  │
  ├─ 系统提示（含参考资料）
  └─ 用户问题
  │
  ▼
调用LLM（流式）
  │
  ▼
┌─────────────────────────────────────┐
│         SSE 事件流                   │
│  data: {"request_id": "xxx"}        │
│  data: {"content": "根据"}          │
│  data: {"content": "参考"}          │
│  data: {"content": "资料"}          │
│  ...                                │
│  data: {"done": true, "token_usage": {...}}│
│  data: [DONE]                       │
└─────────────────────────────────────┘
  │
  ▼
前端解析 SSE 并实时展示
```

## 6. 关键特性

### 6.1 RAG检索增强

- **自动检索**: 生成前自动从选定 Collection 检索相关文档
- **多知识库支持**: 可同时搜索多个 Collection
- **Top-K配置**: 可配置检索返回的文档数量
- **相似度过滤**: 自动过滤低相似度结果
- **Token 截断**: 按模型上下文预算自动裁剪上下文，避免超限

### 6.2 生成配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 模型 | 选择 | deepseek-v3.2 | 选择LLM模型 |
| 温度 | 滑块 | 0.7 | 控制生成随机性，0-2 |
| 最大Token | 滑块 | 4096 | 最大输出长度，1-8192 |
| 知识库 Collection | 多选 | - | 选择检索的知识库 |
| Top-K | 滑块 | 5 | 检索返回数量，1-20 |

### 6.3 引用标注

- **自动标注**: LLM在回答中自动标注引用 [1], [2]
- **来源展示**: 展示引用来源的文件名和相似度
- **原文展示**: 展示引用的原始文本内容
- **高亮显示**: 引用标记在结果中高亮显示

### 6.4 流式输出

- **实时反馈**: 生成内容实时展示，无需等待完成
- **打字机效果**: 模拟打字机效果，提升用户体验
- **可取消**: 支持中途取消生成
- **Token统计**: 完成后显示Token使用量
- **POST + SSE**: 后端返回 SSE 格式，前端使用 `fetch` 解析

### 6.5 历史记录管理

- **自动保存**: 每次生成自动保存历史
- **最大记录数**: 100条
- **自动清理**: 超出限制时自动删除最旧记录
- **详情查看**: 支持查看历史记录的完整内容
- **全部清空**: 支持一键清空全部历史（软删除）

| 场景 | 上下文长度 | 输出长度 | 预期耗时 |
|------|-----------|---------|---------|
| 简单问答 | <1000 tokens | <500 tokens | 2-5s |
