## 1. 向量数据库概述

### 1.1 什么是向量数据库

向量数据库是专门用于存储和检索高维向量数据的数据库系统。它们针对相似性搜索进行了优化，能够在海量向量中快速找到与查询向量最相似的结果。

### 1.2 系统支持的数据库

| 数据库 | 类型 | 特点 | 适用场景 |
|--------|------|------|----------|
| Milvus | 分布式服务 | 高可用、可扩展、原生混合检索 | 生产环境 |

> **注意**: 当前版本仅支持 Milvus 作为向量数据库提供者。

### 1.3 Provider 架构

```python
# 统一的 Provider 接口
class VectorIndexProvider(ABC):
    @abstractmethod
    async def create_index(self, config: IndexConfig) -> str:
        """创建索引"""
        pass
    
    @abstractmethod
    async def add_vectors(self, index_id: str, vectors: List[List[float]], 
                         metadata: List[dict]) -> int:
        """添加向量"""
        pass

    async def search(self, index_id: str, query_vector: List[float],

        """搜索向量"""
        pass
    
    @abstractmethod
    async def delete_index(self, index_id: str) -> bool:
        """删除索引"""
        pass
```

## 2. Milvus 向量数据库

### 2.1 简介

Milvus 是一个开源的向量数据库，专为海量向量数据的存储和检索而设计。它支持分布式部署，具有高可用性和可扩展性。

### 2.2 核心特点

- **分布式架构**: 支持水平扩展
- **高可用性**: 支持副本和故障转移
- **多种索引**: 支持 FLAT、IVF、HNSW 等
- **混合查询**: 支持向量+标量混合查询
- **云原生**: 支持 Kubernetes 部署

### 2.5 Collection 创建

```python
from pymilvus import Collection, FieldSchema, CollectionSchema, DataType

# 定义字段
fields = [
    FieldSchema(name="id", dtype=DataType.VARCHAR, max_length=36, is_primary=True),
    FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=1024),
    FieldSchema(name="text", dtype=DataType.VARCHAR, max_length=65535),
    FieldSchema(name="metadata", dtype=DataType.JSON)
]

# 创建 Schema
schema = CollectionSchema(fields=fields, description="Document embeddings")

# 创建 Collection
collection = Collection(name="doc_embeddings", schema=schema)

# 创建索引
index_params = {
    "metric_type": "IP",  # Inner Product (余弦相似度)
    "index_type": "HNSW",
    "params": {"M": 16, "efConstruction": 256}
}
collection.create_index(field_name="embedding", index_params=index_params)

# 加载到内存
collection.load()
```

### 2.6 数据操作

```python
# 插入数据
data = [
    ["id1", "id2", "id3"],                    # id
    [[0.1, 0.2, ...], [0.3, 0.4, ...], ...],  # embedding
    ["text1", "text2", "text3"],              # text
    [{"page": 1}, {"page": 2}, {"page": 3}]   # metadata
]
collection.insert(data)

# 搜索
search_params = {"metric_type": "IP", "params": {"ef": 64}}
results = collection.search(
    data=[[0.1, 0.2, ...]],  # 查询向量
    anns_field="embedding",
    param=search_params,
    limit=10,
    output_fields=["text", "metadata"]
)
```

## 4. 索引算法详解

### 4.1 FLAT (暴力搜索)

**原理**: 遍历所有向量，计算与查询向量的距离。

**特点**:
- 精度: 100%
- 速度: O(n)
- 内存: O(n*d)

**适用场景**: 数据量小于10万，对精度要求极高

```python
# FAISS
index = faiss.IndexFlatIP(dimension)

# Milvus
index_params = {
    "index_type": "FLAT",
    "metric_type": "IP"
}
```

### 4.2 HNSW (层次可导航小世界图)

**原理**: 构建多层图结构，通过图遍历快速定位近邻。

**特点**:
- 精度: 95%+
- 速度: O(log n)
- 内存: O(n*M)

**参数**:
- M: 每个节点的连接数（推荐16-64）
- efConstruction: 构建时的搜索范围（推荐100-500）
- efSearch: 搜索时的搜索范围（推荐50-200）

```python
# FAISS
index = faiss.IndexHNSWFlat(dimension, M=32)
index.hnsw.efConstruction = 200
index.hnsw.efSearch = 64

# Milvus
index_params = {
    "index_type": "HNSW",
    "metric_type": "IP",
    "params": {"M": 32, "efConstruction": 200}
}
search_params = {"ef": 64}
```

### 4.3 IVF_FLAT (倒排索引)

**原理**: 将向量空间划分为多个聚类，搜索时只在相关聚类中查找。

**特点**:
- 精度: 90%+
- 速度: O(n/nlist * nprobe)
- 内存: O(n*d)

**参数**:
- nlist: 聚类数量（推荐 sqrt(n) 到 4*sqrt(n)）
- nprobe: 搜索时访问的聚类数（推荐 nlist/10 到 nlist/2）

```python
# FAISS
nlist = 100
quantizer = faiss.IndexFlatIP(dimension)
index = faiss.IndexIVFFlat(quantizer, dimension, nlist, faiss.METRIC_INNER_PRODUCT)
index.train(vectors)  # 需要训练
index.add(vectors)
index.nprobe = 10

# Milvus
index_params = {
    "index_type": "IVF_FLAT",
    "metric_type": "IP",
    "params": {"nlist": 100}
}
search_params = {"nprobe": 10}
```

### 4.4 IVF_PQ (乘积量化)

**原理**: 结合倒排索引和乘积量化，大幅压缩向量存储。

**特点**:
- 精度: 80%+
- 速度: 快
- 内存: O(n*m)（m << d）

**参数**:
- nlist: 聚类数量
- m: 子向量数量（必须能整除维度）
- nbits: 每个子向量的编码位数（通常8）

```python
# FAISS
nlist = 100
m = 8  # 子向量数量
quantizer = faiss.IndexFlatIP(dimension)
index = faiss.IndexIVFPQ(quantizer, dimension, nlist, m, 8)
index.train(vectors)
index.add(vectors)

# Milvus
index_params = {
    "index_type": "IVF_PQ",
    "metric_type": "IP",
    "params": {"nlist": 100, "m": 8, "nbits": 8}
}
```

### 4.5 IVF_SQ8 (标量量化) — 新增

**原理**: 结合倒排索引和 8-bit 标量量化，将每个向量维度从 32-bit 浮点压缩为 8-bit 整数。

**特点**:
- 精度: 93%+（精度损失极小）
- 速度: 快
- 内存: 节省约 75%

**参数**:
- nlist: 聚类数量（推荐 `sqrt(n_vectors)`，范围 1-65536）

```python
# Milvus
index_params = {
    "index_type": "IVF_SQ8",
    "metric_type": "COSINE",
    "params": {"nlist": 128}
}
search_params = {"nprobe": 10}
```

**适用场景**: 50万~500万数据量，追求内存效率与精度平衡

### 4.6 算法对比

| 算法 | 精度 | 构建速度 | 搜索速度 | 内存占用 | 适用数据量 | 智能推荐 |
|------|------|---------|---------|---------|-----------|---------|
| FLAT | 100% | 快 | 慢 | 高 | <1万 | ✅ |
| HNSW | 95%+ | 中 | 快 | 中 | 1万~50万 | ✅ |
| IVF_FLAT | 90%+ | 中 | 中 | 高 | 10万~100万 | - |
| IVF_SQ8 | 93%+ | 中 | 快 | 低（节省75%） | 50万~500万 | ✅ |
| IVF_PQ | 80%+ | 慢 | 快 | 极低 | >500万 | ✅ |

## 5. 度量类型说明

### 5.1 余弦相似度 (Cosine Similarity)

**公式**: 
$$\cos(\theta) = \frac{A \cdot B}{||A|| \times ||B||}$$

**特点**:
- 值域: [-1, 1]
- 不受向量长度影响
- 适合文本语义匹配

**实现方式**:
```python
# 归一化后使用内积
faiss.normalize_L2(vectors)
index = faiss.IndexFlatIP(dimension)
```

### 5.2 欧氏距离 (Euclidean Distance / L2)

**公式**: 
$$d(A, B) = \sqrt{\sum_{i=1}^{n}(A_i - B_i)^2}$$

**特点**:
- 值域: [0, ∞)
- 距离越小越相似
- 适合图像特征匹配

**实现方式**:
```python
index = faiss.IndexFlatL2(dimension)
```

### 5.3 点积 (Inner Product / Dot Product)

**公式**: 
$$A \cdot B = \sum_{i=1}^{n}A_i \times B_i$$

**特点**:
- 值域: (-∞, +∞)
- 受向量长度影响
- 适合归一化向量

**实现方式**:
```python
index = faiss.IndexFlatIP(dimension)
```

### 5.4 度量类型选择

| 度量类型 | Milvus 映射 | 代码值 | 适用场景 | 自动推荐模型 |
|---------|-----------|--------|----------|------------|
| 余弦相似度 | COSINE | cosine | 文本语义 | BGE/Qwen/OpenAI/Cohere |
| 欧氏距离 | L2 | euclidean | 图像特征 | 未识别模型（默认） |
| 点积 | IP | dot_product | 归一化向量 | - |

**度量类型自动推断规则**:
```python
METRIC_TYPE_RULES = {
    "bge": "COSINE",       # BGE 系列模型
    "qwen": "COSINE",      # Qwen 系列（qwen3-embedding-8b 等）
    "openai": "COSINE",    # OpenAI 系列
    "cohere": "COSINE",    # Cohere 系列
    "text-embedding": "COSINE",  # OpenAI text-embedding-xxx
    "ada": "COSINE",       # OpenAI ada
    "default": "L2",       # 未识别模型的安全默认值
}
```

## 6. Milvus Collection 管理（Dify 方案）

### 6.1 「逻辑与物理分层」架构

采用参考 Dify 的「逻辑与物理分层」架构，彻底解决 Milvus 不支持同一 Collection 存储不同维度向量的问题：

```
逻辑知识库: collection_name
├── 物理 Collection: default_collection_dim1024  (1024维向量)
├── 物理 Collection: default_collection_dim2048  (2048维向量)
└── 物理 Collection: default_collection_dim4096  (4096维向量)
```

### 6.3 Collection Schema

```python
# 每个物理 Collection 的 Schema
fields = [
    FieldSchema(name="id", dtype=VARCHAR, is_primary=True, max_length=100),
    FieldSchema(name="doc_id", dtype=VARCHAR, max_length=256,
                is_partition_key=True,  # Partition Key: Milvus 自动按文档ID哈希分区
                description="文档ID"),
    FieldSchema(name="embedding", dtype=FLOAT_VECTOR, dim=dimension,
                description="稠密向量（固定维度）"),
    FieldSchema(name="sparse_embedding", dtype=SPARSE_FLOAT_VECTOR,
                description="稀疏向量（BM25 生成）"),  # 可选字段
    FieldSchema(name="metadata", dtype=JSON)
]
```

### 6.4 优势

1. **消除维度冲突**: 每个物理 Collection 只有一个固定维度的 embedding 字段
2. **零向量填充**: 无需为其他维度填充零向量，节省存储空间
3. **独立优化**: 每个物理 Collection 可独立优化索引参数
4. **模型迭代**: 切换嵌入模型时旧 Collection 保留，天然支持版本迭代

## 8. 最佳实践

### 8.1 索引选择建议（智能推荐引擎规则）

| 数据量 | 推荐索引 | 参数建议 | 推荐理由 |
|--------|---------|----------|---------|
| <1万 | FLAT | - | 暴力搜索保证100%精确召回 |
| 1万~50万 | HNSW | M=16, efConstruction=200 | 图索引兼顾高召回率与低延迟 |
| 50万~500万 | IVF_SQ8 | nlist=128 | 标量量化在精度损失极小下节省75%内存 |
| >500万 | IVF_PQ | nlist=1024, m=8, nbits=8 | 乘积量化大幅压缩内存占用 |

### 8.2 性能优化建议

1. **使用智能推荐**: 优先采用推荐引擎的建议（采纳率目标 ≥ 80%）
2. **启用混合检索**: 对中文文档启用 BM25 稀疏向量，提升关键词召回（详见 [检索查询模块](../search-query/01-检索查询功能概述.md)）
3. **批量操作**: 批量插入，减少IO开销
4. **Partition Key**: 利用 doc_id 作为 Partition Key，提升按文档过滤的查询性能
5. **参数调优**: 根据精度-速度权衡调整参数

### 8.3 运维建议

1. **监控推荐采纳率**: 通过 `GET /api/vector-index/recommend/stats` 追踪推荐引擎效果
2. **检查 Reranker 状态**: 通过 `GET /api/v1/search/reranker/health` 确认 Reranker 可用性（详见 [检索查询模块 API](../search-query/02-API接口文档.md)）
3. **容量规划**: 提前规划 Milvus 集群存储和内存需求
4. **Collection 管理**: 通过 `GET /api/vector-index/collections` 监控各 Collection 的文档数和向量数
