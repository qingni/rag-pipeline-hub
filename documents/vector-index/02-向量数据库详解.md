## 1. 向量数据库概述

### 1.1 什么是向量数据库

向量数据库是专门用于存储和检索高维向量数据的数据库系统。它们针对相似性搜索进行了优化，能够在海量向量中快速找到与查询向量最相似的结果。

### 1.2 系统支持的数据库

| 数据库 | 类型 | 特点 | 适用场景 |
|--------|------|------|----------|
| Milvus | 分布式服务 | 高可用、可扩展 | 生产环境 |
| FAISS | 本地库 | 轻量、高性能 | 开发测试 |

### 1.3 Provider 架构

```python
# 统一的 Provider 接口
class VectorIndexProvider(ABC):
    @abstractmethod
    async def create_index(self, config: IndexConfig) -> str:
        """创建索引"""
        pass
    
    @abstractmethod
    async def add_vectors(self, index_id: str, vectors: List[List[float]], 
                         metadata: List[dict]) -> int:
        """添加向量"""
        pass
    
    @abstractmethod
    async def search(self, index_id: str, query_vector: List[float], 
                    top_k: int) -> List[SearchResult]:
        """搜索向量"""
        pass
    
    @abstractmethod
    async def delete_index(self, index_id: str) -> bool:
        """删除索引"""
        pass
```

## 2. Milvus 向量数据库

### 2.1 简介

Milvus 是一个开源的向量数据库，专为海量向量数据的存储和检索而设计。它支持分布式部署，具有高可用性和可扩展性。

### 2.2 核心特点

- **分布式架构**: 支持水平扩展
- **高可用性**: 支持副本和故障转移
- **多种索引**: 支持 FLAT、IVF、HNSW 等
- **混合查询**: 支持向量+标量混合查询
- **云原生**: 支持 Kubernetes 部署

### 2.5 Collection 创建

```python
from pymilvus import Collection, FieldSchema, CollectionSchema, DataType

# 定义字段
fields = [
    FieldSchema(name="id", dtype=DataType.VARCHAR, max_length=36, is_primary=True),
    FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=1024),
    FieldSchema(name="text", dtype=DataType.VARCHAR, max_length=65535),
    FieldSchema(name="metadata", dtype=DataType.JSON)
]

# 创建 Schema
schema = CollectionSchema(fields=fields, description="Document embeddings")

# 创建 Collection
collection = Collection(name="doc_embeddings", schema=schema)

# 创建索引
index_params = {
    "metric_type": "IP",  # Inner Product (余弦相似度)
    "index_type": "HNSW",
    "params": {"M": 16, "efConstruction": 256}
}
collection.create_index(field_name="embedding", index_params=index_params)

# 加载到内存
collection.load()
```

### 2.6 数据操作

```python
# 插入数据
data = [
    ["id1", "id2", "id3"],                    # id
    [[0.1, 0.2, ...], [0.3, 0.4, ...], ...],  # embedding
    ["text1", "text2", "text3"],              # text
    [{"page": 1}, {"page": 2}, {"page": 3}]   # metadata
]
collection.insert(data)

# 搜索
search_params = {"metric_type": "IP", "params": {"ef": 64}}
results = collection.search(
    data=[[0.1, 0.2, ...]],  # 查询向量
    anns_field="embedding",
    param=search_params,
    limit=10,
    output_fields=["text", "metadata"]
)
```

## 3. FAISS 本地索引

### 3.1 简介

FAISS (Facebook AI Similarity Search) 是 Facebook 开发的高效相似性搜索库。它是一个本地库，不需要服务端部署，适合开发测试和中小规模数据。

### 3.2 核心特点

- **高性能**: CPU/GPU 优化
- **轻量级**: 无需服务端
- **多种索引**: 支持多种索引类型
- **内存高效**: 支持向量压缩
- **易于集成**: Python 接口简洁

### 3.3 索引类型

| 索引类型 | 说明 | 适用场景 |
|---------|------|----------|
| IndexFlatL2 | 精确L2距离 | 小数据量 |
| IndexFlatIP | 精确内积 | 小数据量 |
| IndexIVFFlat | 倒排索引 | 中等数据量 |
| IndexIVFPQ | 乘积量化 | 大数据量 |
| IndexHNSWFlat | 图索引 | 低延迟要求 |

### 3.4 使用示例

```python
import faiss
import numpy as np

# 创建索引
dimension = 1024
index = faiss.IndexFlatIP(dimension)  # 内积索引

# 添加向量
vectors = np.random.random((1000, dimension)).astype('float32')
faiss.normalize_L2(vectors)  # 归一化（用于余弦相似度）
index.add(vectors)

# 搜索
query = np.random.random((1, dimension)).astype('float32')
faiss.normalize_L2(query)
distances, indices = index.search(query, k=10)

print(f"Top 10 indices: {indices[0]}")
print(f"Top 10 distances: {distances[0]}")
```

### 3.5 HNSW 索引示例

```python
import faiss

# 创建 HNSW 索引
dimension = 1024
M = 32  # 每个节点的连接数
index = faiss.IndexHNSWFlat(dimension, M, faiss.METRIC_INNER_PRODUCT)

# 设置构建参数
index.hnsw.efConstruction = 200

# 添加向量
index.add(vectors)

# 设置搜索参数
index.hnsw.efSearch = 64

# 搜索
distances, indices = index.search(query, k=10)
```

```python
# 保存索引
faiss.write_index(index, "index.faiss")

# 加载索引
index = faiss.read_index("index.faiss")
```

## 4. 索引算法详解

### 4.1 FLAT (暴力搜索)

**原理**: 遍历所有向量，计算与查询向量的距离。

**特点**:
- 精度: 100%
- 速度: O(n)
- 内存: O(n*d)

**适用场景**: 数据量小于10万，对精度要求极高

```python
# FAISS
index = faiss.IndexFlatIP(dimension)

# Milvus
index_params = {
    "index_type": "FLAT",
    "metric_type": "IP"
}
```

### 4.2 HNSW (层次可导航小世界图)

**原理**: 构建多层图结构，通过图遍历快速定位近邻。

**特点**:
- 精度: 95%+
- 速度: O(log n)
- 内存: O(n*M)

**参数**:
- M: 每个节点的连接数（推荐16-64）
- efConstruction: 构建时的搜索范围（推荐100-500）
- efSearch: 搜索时的搜索范围（推荐50-200）

```python
# FAISS
index = faiss.IndexHNSWFlat(dimension, M=32)
index.hnsw.efConstruction = 200
index.hnsw.efSearch = 64

# Milvus
index_params = {
    "index_type": "HNSW",
    "metric_type": "IP",
    "params": {"M": 32, "efConstruction": 200}
}
search_params = {"ef": 64}
```

### 4.3 IVF_FLAT (倒排索引)

**原理**: 将向量空间划分为多个聚类，搜索时只在相关聚类中查找。

**特点**:
- 精度: 90%+
- 速度: O(n/nlist * nprobe)
- 内存: O(n*d)

**参数**:
- nlist: 聚类数量（推荐 sqrt(n) 到 4*sqrt(n)）
- nprobe: 搜索时访问的聚类数（推荐 nlist/10 到 nlist/2）

```python
# FAISS
nlist = 100
quantizer = faiss.IndexFlatIP(dimension)
index = faiss.IndexIVFFlat(quantizer, dimension, nlist, faiss.METRIC_INNER_PRODUCT)
index.train(vectors)  # 需要训练
index.add(vectors)
index.nprobe = 10

# Milvus
index_params = {
    "index_type": "IVF_FLAT",
    "metric_type": "IP",
    "params": {"nlist": 100}
}
search_params = {"nprobe": 10}
```

### 4.4 IVF_PQ (乘积量化)

**原理**: 结合倒排索引和乘积量化，大幅压缩向量存储。

**特点**:
- 精度: 80%+
- 速度: 快
- 内存: O(n*m)（m << d）

**参数**:
- nlist: 聚类数量
- m: 子向量数量（必须能整除维度）
- nbits: 每个子向量的编码位数（通常8）

```python
# FAISS
nlist = 100
m = 8  # 子向量数量
quantizer = faiss.IndexFlatIP(dimension)
index = faiss.IndexIVFPQ(quantizer, dimension, nlist, m, 8)
index.train(vectors)
index.add(vectors)

# Milvus
index_params = {
    "index_type": "IVF_PQ",
    "metric_type": "IP",
    "params": {"nlist": 100, "m": 8, "nbits": 8}
}
```

### 4.5 算法对比

| 算法 | 精度 | 构建速度 | 搜索速度 | 内存占用 | 适用数据量 |
|------|------|---------|---------|---------|-----------|
| FLAT | 100% | 快 | 慢 | 高 | <10万 |
| HNSW | 95%+ | 中 | 快 | 中 | 10万-1000万 |
| IVF_FLAT | 90%+ | 中 | 中 | 高 | 10万-1000万 |
| IVF_PQ | 80%+ | 慢 | 快 | 低 | >1000万 |

## 5. 度量类型说明

### 5.1 余弦相似度 (Cosine Similarity)

**公式**: 
$$\cos(\theta) = \frac{A \cdot B}{||A|| \times ||B||}$$

**特点**:
- 值域: [-1, 1]
- 不受向量长度影响
- 适合文本语义匹配

**实现方式**:
```python
# 归一化后使用内积
faiss.normalize_L2(vectors)
index = faiss.IndexFlatIP(dimension)
```

### 5.2 欧氏距离 (Euclidean Distance / L2)

**公式**: 
$$d(A, B) = \sqrt{\sum_{i=1}^{n}(A_i - B_i)^2}$$

**特点**:
- 值域: [0, ∞)
- 距离越小越相似
- 适合图像特征匹配

**实现方式**:
```python
index = faiss.IndexFlatL2(dimension)
```

### 5.3 点积 (Inner Product / Dot Product)

**公式**: 
$$A \cdot B = \sum_{i=1}^{n}A_i \times B_i$$

**特点**:
- 值域: (-∞, +∞)
- 受向量长度影响
- 适合归一化向量

**实现方式**:
```python
index = faiss.IndexFlatIP(dimension)
```

### 5.4 度量类型选择

| 度量类型 | Milvus | FAISS | 适用场景 |
|---------|--------|-------|----------|
| 余弦相似度 | IP (归一化) | IP (归一化) | 文本语义 |
| 欧氏距离 | L2 | L2 | 图像特征 |
| 点积 | IP | IP | 归一化向量 |

## 6. 数据库选择指南

### 6.1 选择 Milvus 的场景

- ✅ 生产环境部署
- ✅ 数据量超过100万
- ✅ 需要高可用性
- ✅ 需要分布式扩展
- ✅ 需要混合查询（向量+标量）
- ✅ 团队有运维能力

### 6.2 选择 FAISS 的场景

- ✅ 开发测试环境
- ✅ 数据量小于100万
- ✅ 单机部署
- ✅ 快速原型验证
- ✅ 对延迟要求极高
- ✅ 资源有限

## 7. 最佳实践

### 7.1 索引选择建议

| 数据量 | 推荐索引 | 参数建议 |
|--------|---------|----------|
| <1万 | FLAT | - |
| 1万-10万 | HNSW | M=16, ef=200 |
| 10万-100万 | HNSW/IVF_FLAT | M=32, ef=256 / nlist=1024 |
| >100万 | IVF_PQ | nlist=4096, m=8 |

### 7.2 性能优化建议

1. **向量归一化**: 使用余弦相似度时，预先归一化向量
2. **批量操作**: 批量插入和搜索，减少IO开销
3. **内存管理**: 大数据量时使用磁盘索引或分片
4. **参数调优**: 根据精度-速度权衡调整参数

### 7.3 运维建议

1. **定期备份**: 定期备份索引数据
2. **监控告警**: 监控查询延迟和错误率
3. **容量规划**: 提前规划存储和内存需求
4. **版本管理**: 记录索引配置和版本信息
