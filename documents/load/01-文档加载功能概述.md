## 1. 功能概述

### 1.1 核心功能
文档加载功能是RAG (Retrieval-Augmented Generation) 系统的首要环节,负责将上传的文档解析为可处理的文本数据,为后续的分块、向量化和检索提供基础。

### 1.2 主要特性
- ✅ **多格式支持**: PDF、DOCX、DOC、TXT、Markdown
- ✅ **多加载器**: 针对不同格式提供多种加载器选择
- ✅ **自动选择**: 智能匹配最佳加载器
- ✅ **元数据提取**: 提取文档标题、作者、页数等信息
- ✅ **按页解析**: 保留页面结构信息
- ✅ **文档预览**: 实时查看上传文档内容
- ✅ **结果管理**: 查看、下载加载结果

### 1.3 设计原则
1. **模块化**: 各加载器独立实现,易于扩展
2. **持久化**: 结果保存为JSON格式,支持回溯查看
3. **容错性**: 多层级降级策略,确保加载成功率
4. **性能优化**: 异步处理、进度反馈、结果缓存

## 5. 工作流程

### 5.1 完整加载流程

```
1. 用户上传文档
   └─> DocumentUploader.vue
       └─> POST /api/documents/upload
           └─> 保存到 uploads/ 目录
           └─> 创建 Document 记录

2. 系统自动选择加载器
   └─> loading_service.load_document()
       └─> 根据文件格式选择默认加载器
           ├─ PDF → pymupdf
           ├─ DOCX → docx
           ├─ DOC → doc
           └─ TXT/MD → text

3. 执行文档加载
   └─> loader.extract_text()
       ├─> 提取文本内容
       ├─> 提取元数据
       ├─> 按页组织数据
       └─> 统计信息

4. 保存加载结果
   └─> json_storage.save_result()
       └─> 保存到 backend/results/load/
       └─> 创建 ProcessingResult 记录

5. 前端展示结果
   └─> ResultPreview.vue
       ├─> 显示元数据
       ├─> 显示文本内容
       └─> 提供下载功能
```

### 5.2 加载器选择策略

```
用户未指定加载器
  │
  ▼
检查文件格式
  │
  ├─ PDF
  │  └─> pymupdf (默认)
  │      └─> 失败则降级到 pypdf
  │
  ├─ DOCX
  │  └─> docx
  │
  ├─ DOC
  │  └─> doc (antiword/python-docx)
  │
  └─ TXT/MD
     └─> text
```

## 6. 关键特性

### 6.1 格式自动识别
系统根据文件扩展名自动选择合适的加载器,无需用户手动指定。

### 6.2 多加载器支持
针对PDF格式提供三种加载器可选:
- **PyMuPDF**: 性能最佳,功能全面(默认)
- **PyPDF2**: 轻量级,纯Python实现
- **Unstructured**: 支持复杂布局和表格

### 6.3 元数据提取
自动提取文档元数据:
- 标题、作者、主题
- 创建时间、修改时间
- 页数、文件大小
- 创建工具、格式版本

### 6.4 结构化输出
加载结果保持文档结构:
- 按页组织文本内容
- 保留页码信息
- 统计每页字符数
- 生成完整文本

### 6.5 错误处理
完善的错误处理机制:
- 格式不支持提示
- 文件损坏检测
- 加载失败降级
- 详细错误信息
