## 1. 功能概述

### 1.1 核心功能
文档加载功能是RAG (Retrieval-Augmented Generation) 系统的首要环节,负责将上传的文档加载为可处理的文本数据,为后续的分块、向量化和检索提供基础。系统采用 Docling Serve 作为主解析器（HTTP 服务模式），并支持自动降级到专用加载器。

### 1.2 主要特性
- ✅ **多格式支持**: PDF、DOCX、DOC、XLSX、XLS、PPTX、HTML、CSV、JSON、XML、TXT、MD 等 15+ 种格式
- ✅ **Docling Serve 集成**: 采用 Docling Serve HTTP 服务作为主解析器，支持异步模式，避免启动时初始化延迟
- ✅ **异步加载**: 大文件和复杂文档使用异步任务队列处理，支持进度追踪和任务管理
- ✅ **多加载器**: 针对不同格式提供多种加载器选择
- ✅ **自动选择**: 智能匹配最佳加载器
- ✅ **自动降级**: 主解析器失败时自动降级到备用解析器
- ✅ **元数据提取**: 提取文档标题、作者、页数等信息
- ✅ **按页加载**: 保留页面结构信息
- ✅ **表格提取**: 支持表格数据结构化提取（Markdown 格式）
- ✅ **图片提取优化**: 采用占位符+结构化元数据方案，支持 base64 数据用于多模态嵌入
- ✅ **多格式输出**: Markdown（预览）、结构化文本（RAG）、纯文本（兼容）
- ✅ **加载结果持久化**: 结果保存为 JSON 文件，支持预览和分块功能复用
- ✅ **文档预览**: 实时查看上传文档内容（含图片），支持四种预览状态
- ✅ **结果管理**: 查看、下载加载结果，展示加载耗时和解析器信息

### 1.3 设计原则
1. **模块化**: 各加载器独立实现,易于扩展
2. **持久化**: 结果保存为JSON格式,支持回溯查看
3. **容错性**: 多层级降级策略,确保加载成功率
4. **性能优化**: 异步处理、进度反馈、结果缓存

## 5. 工作流程

### 5.1 完整加载流程

```
1. 用户上传文档
   └─> DocumentUploader.vue
       └─> POST /api/v1/documents
           └─> 保存到 uploads/ 目录
           └─> 创建 Document 记录（status = uploaded）

2. 用户或系统触发文档加载
   └─> loading_service.load_document() 或通过 /api/v1/processing/load / /api/v1/processing/load/async
       └─> fallback_manager 根据文件格式和大小选择格式策略
           ├─ Docling Serve 作为 PDF/DOCX/XLSX/PPTX 等复杂格式的主解析器
           ├─ Docling 失败或不可用时自动降级到专用加载器（pymupdf/docx/xlsx/pptx/...）
           └─ 对 HTML/JSON/TXT/MD 等格式直接使用对应专用加载器

3. 执行文档加载
   └─> 具体 loader.extract_text()
       ├─> Docling Serve: 提交异步任务（/v1/convert/file/async），返回 async_mode + task_id
       ├─> 其他加载器: 同步解析文件
       ├─> 提取文本内容（full_text + pages）
       ├─> 提取元数据（页数、标题、作者、图片/表格数量等）
       └─> 统计信息（total_pages/total_chars/processing_time 等）

4. 保存加载结果
   └─> json_storage.save_result()
       └─> 保存到 backend/results/load/ 目录
       └─> 创建 ProcessingResult 记录（processing_type = "load"，status = completed/running/failed）
       └─> 异步模式下同时创建 LoadingTask 记录，用于追踪 Docling Serve 任务状态和进度
       └─> 更新 Document.status = ready / error，并记录处理耗时

5. 前端展示结果
   ├─> DocumentList.vue：在文档列表中展示最近一次加载耗时和解析器（loading_time / provider）
   ├─> DocumentPreview.vue：通过 GET /api/v1/documents/{id}/preview 展示文档预览状态
   └─> ResultPreview.vue：通过 ProcessingResult + JSON 结果展示元数据、文本内容和统计信息，并提供下载功能
```
### 5.2 加载器选择策略

```
用户未指定加载器
  │
  ▼
检查 Docling Serve 是否可用（HTTP 健康检查）
  │
  ├─ 可用 → 使用 Docling Serve 作为主解析器
  │         ├─> 支持同步/异步两种模式
  │         ├─> 支持 OCR（中文+英文）
  │         ├─> 自动提取表格、图片
  │         └─> 失败则自动降级到专用加载器
  │
  └─ 不可用 → 根据文件格式选择专用加载器
             │
             ├─ PDF → pymupdf (默认)
             │        └─> 失败则降级到 pypdf
             │
             ├─ DOCX → docx
             │
             ├─ XLSX/XLS → xlsx
             │
             ├─ PPTX → pptx
             │
             ├─ HTML → html（增强图片提取）
             │
             ├─ JSON → json（结构化文本提取）
             │
             ├─ CSV → csv
             │
             └─ TXT/MD → text
```

## 6. 关键特性

### 6.1 格式自动识别
系统根据文件扩展名自动选择合适的加载器,无需用户手动指定。

### 6.2 Docling Serve 主解析器
采用独立部署的 Docling Serve HTTP 服务作为主解析器，目前在本项目中**只使用异步模式**：
- **异步任务提交**: 调用 `/v1/convert/file/async` 提交转换任务，立即返回 `task_id`
- **任务状态查询**: 通过 `/v1/status/poll/{task_id}` 轮询任务状态（pending/started/success/failure）
- **结果获取**: 任务成功后，通过 `/v1/result/{task_id}` 获取完整解析结果，由 `DoclingServeLoader._parse_response` 转换为标准化结构
- **异步集成方式**: `DoclingServeLoader.extract_text` 负责提交异步任务并返回 `async_mode=true + task_id`，上层 `LoadingService` 通过 `submit_async_load` / `get_async_task_status` / `get_async_task_result` 完成结果落盘与状态更新
- **OCR 支持**: 内置 RapidOCR，支持中文和英文识别
- **表格识别**: 自动识别和结构化提取表格数据
- **图片提取**: 支持嵌入式图片导出（Base64）并在文本中插入 `[IMAGE_N: 描述]` 占位符，便于后续分块和多模态检索

### 6.3 多加载器支持
针对PDF格式提供三种加载器可选:
- **PyMuPDF**: 性能最佳,功能全面(默认降级)
- **PyPDF2**: 轻量级,纯Python实现
- **Unstructured**: 支持复杂布局和表格

### 6.4 元数据提取
自动提取文档元数据:
- 标题、作者、主题
- 创建时间、修改时间
- 页数、文件大小
- 创建工具、格式版本
- Excel Sheet 名称列表

### 6.5 图片提取优化（HTML）
HTMLLoader 针对图片提取进行了全面优化：

**多源图片URL支持**:
- 标准 `src` 属性
- 懒加载 `data-src`、`data-lazy-src`
- 响应式 `srcset` 解析

**完整图片元数据**:
- 基本属性：src、alt、title、尺寸、CSS类、ID
- 高级属性：srcset、loading策略、图片类型
- 上下文信息：父元素、相邻文本（前后各100字符）

**智能图片标题查找**:
- `<figcaption>` 标签
- `title` 属性
- `aria-label` 属性
- `aria-describedby` 引用元素

**三种文本格式输出**:
- `full_text` (Markdown): 包含 `![alt](src)` 格式，支持前端图片预览
- `structured_text` (RAG优化): 包含 `[IMAGE_0: src="..." alt="..." context="..."]` 标记
- `plain_text` (纯文本): 向后兼容的纯文本内容

### 6.6 JSON 结构化文本提取
JSONLoader 优化了文本内容提取：
- 递归提取所有字符串值
- 保留键值关系（`key: value` 格式）
- 自动识别数组/对象结构
- 表格化数组对象数据

### 6.7 结构化输出
加载结果保持文档结构:
- 按页组织文本内容
- 保留页码信息
- 统计每页字符数
- 生成完整文本
- Excel 按 Sheet 分页

### 6.8 错误处理
完善的错误处理机制:
- 格式不支持提示
- 文件损坏检测
- 加载失败降级
- 详细错误信息
- 服务可用性缓存（带过期重检）
