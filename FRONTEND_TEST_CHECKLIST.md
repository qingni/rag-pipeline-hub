## 测试数据准备

### 前置条件
1. 数据库中至少有3个已上传的文档
2. 至少1个文档已完成分块（有 ChunkingResult）
3. 至少1个文档有多个分块结果（不同策略）
4. 后端 embedding API 配置正确（.env 文件）

### 测试场景数据
- **场景1：正常流程**
  - 文档A：已分块，使用 semantic 策略，50个分块
  - 预期：全部成功向量化

- **场景2：部分失败**
  - 文档B：已分块，100个分块
  - 模拟：部分分块文本为空或超长
  - 预期：显示部分成功状态和错误列表

- **场景3：完全失败**
  - 使用错误的 API 配置
  - 预期：显示失败状态和错误信息
