我要实现一个**文档处理和检索系统**，其核心功能模块如下：

1. **文档加载（LoadFile）**
   - 支持多种加载方式（PyMuPDF、PyPDF、Unstructured）
   - 包含文件上传处理、文档预览与管理

2. **文档分块（ChunkFile）**
   - 支持多种分块策略
   - 包含块内容预览、分块文档管理

3. **文档解析（ParseFile）**
   - 支持多种解析选项（全文、分页、按标题等）
   - 包含解析结果预览，支持表格与文本混合解析

4. **向量嵌入（EmbeddingFile）**
   - 支持多个嵌入提供商（OpenAI、Bedrock、HuggingFace）
   - 包含嵌入结果管理、嵌入可视化

5. **向量索引（Indexing）**
   - 支持多种向量数据库（Milvus、Pinecone等）
   - 包含索引管理、集合操作接口、相似度搜索（Search）、语义搜索
   - 支持结果过滤排序、搜索结果保存

6. **文本生成（Generation）**
   - 支持多个生成模型
   - 包含上下文感知生成、结果保存功能

## 前端框架设计
### 技术栈和项目结构
- Vue3 + Vite 作为基础开发框架
- TailwindCSS 用于样式管理
- Vue Router 4 用于路由管理

## 用户界面设计
### 导航系统
- 左侧固定的导航栏，清晰的功能模块分类。
- 选择一个功能模块之后，右边就显示相关的功能模块页面。
### 组件化设计
- 主功能模块页面要求统一的页面布局，左侧控制面板 + 右侧内容显示的布局模式。
- 使用 Grid 系统进行响应式布局，状态管理。
- 使用 Vue 3 Composition API 进行状态管理，统一的错误处理和状态提示，
- API 交互采用统一的后端接口调用模式，错误处理和加载状态管理。


## 后端技术栈
- 通过FastAPI实现。
- 涉及到AI模型，目前可以给我Ollama和HuggingFace两个选项。
- 每一个功能模块单独写入相关的service文件，如loading_service.py; embedding_service.py。
- 每一个功能的实现结果都保存进入一个json，文件名就是文档名_时戳，供下一步调用。
