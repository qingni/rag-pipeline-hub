## 🎯 演示场景

### 场景1: 智能加载器自动切换

**演示步骤**:

1. **上传PDF文档**
   ```
   操作: 拖拽一个 PDF 文件到上传区域
   观察: 文件上传完成
   ```

2. **查看加载器自动切换**
   ```
   期望: 加载器自动切换到 "PyMuPDF"
   说明: 系统识别到 PDF 格式，自动选择最佳加载器
   ```

3. **上传DOCX文档**
   ```
   操作: 上传一个 DOCX 文件
   观察: 加载器自动切换到 "DOCX"
   ```

4. **上传Markdown文件**
   ```
   操作: 上传一个 .md 文件
   观察: 加载器自动切换到 "Text/Markdown"
   ```

**演示要点**:
- ✨ 强调"自动"特性 - 无需手动操作
- 💡 说明每种格式的最佳加载器
- 🎯 展示手动调整的灵活性

### 场景2: 表格视图高效浏览

**演示步骤**:

1. **准备数据**
   ```
   确保已上传 5-10 个不同格式的文档
   ```

2. **展示紧凑布局**
   ```
   对比点:
   - 一屏可以看到更多文档
   - 所有关键信息清晰可见
   - 文件图标帮助快速识别
   ```

3. **演示交互效果**
   ```
   操作: 鼠标悬停在不同文档行上
   观察: 行背景变化，清晰的视觉反馈
   
   操作: 点击某一行
   观察: 选中行变为蓝色背景
   ```

4. **展示智能时间显示**
   ```
   观察: 
   - 刚上传的: "刚刚"
   - 几分钟前的: "5分钟前"
   - 更早的: "2小时前" 或具体日期时间
   ```

**演示要点**:
- 📊 对比优化前后的空间利用率
- 🎨 强调清晰的视觉层次
- ⚡ 展示流畅的交互体验

### 场景3: 便捷的文档删除

**演示步骤**:

1. **删除单个文档**
   ```
   操作: 点击某个文档行右侧的"删除"按钮
   观察: 弹出确认对话框
   ```

2. **确认对话框**
   ```
   观察内容:
   - 清晰的标题: "确认删除"
   - 文档名称显示
   - 警告信息: "此操作不可撤销..."
   - 两个按钮: "取消" 和 "确认删除"
   ```

3. **执行删除**
   ```
   操作: 点击"确认删除"
   观察: 
   - 按钮显示"删除中..."
   - 文档从列表中移除
   - 如果是当前选中的文档，右侧详情自动清空
   ```

4. **智能分页处理**
   ```
   场景: 删除当前页的最后一个文档
   观察: 自动跳转到上一页
   ```

**演示要点**:
- 🛡️ 强调安全机制 - 二次确认
- 🧹 展示清理的完整性 - 删除相关数据
- 🎯 说明智能的分页处理

### 场景4: 完整工作流程

**端到端演示**:

```
1. 上传文档 (document.docx)
   ↓ 自动出现在列表中，带 📝 图标
   
2. 自动选择文档
   ↓ 加载器自动切换到 "DOCX"
   
3. (可选) 调整加载器
   ↓ 例如切换到 "Unstructured"
   
4. 点击"开始加载"
   ↓ 显示加载进度
   
5. 查看加载结果
   ↓ 右侧显示文档内容预览
   
6. 不需要时删除
   ↓ 点击删除 → 确认 → 完成
```

**时间**: 约 2-3 分钟

**演示要点**:
- 🎯 展示完整的用户旅程
- ⚡ 强调操作的流畅性
- 💡 说明各个功能如何协同工作

## 🎯 常见问题准备

### Q1: 自动切换会不会限制用户选择？
```
A: 不会。自动切换只是提供智能的默认选项，
   用户随时可以手动调整为其他加载器。
   这种设计既降低了使用门槛，又保留了灵活性。
```

### Q2: 表格视图在移动端如何显示？
```
A: 表格支持横向滚动，在移动端可以左右滑动查看所有列。
   我们保留了核心功能，确保移动端也能正常使用。
```

### Q3: 删除操作可以撤销吗？
```
A: 不能。这是为了确保数据的一致性。
   所以我们设计了确认对话框，防止误操作。
   未来可能会考虑回收站功能。
```

### Q4: 如何批量删除文档？
```
A: 当前版本支持单个删除。
   批量删除功能已经在计划中，将在下一个版本推出。
```
